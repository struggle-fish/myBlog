# 目录 



一、核心基础：Options API vs Composition API

`Options API`：Vue2 传统写法，按 `data/methods/computed/watch` 等选项组织代码，适合简单场景。

`Composition API`：Vue3 推荐写法，用 setup 作为入口，通过组合函数`（ref/reactive/computed 等）`组织逻辑，更灵活、可复用性更强。


二、响应式核心 API

- ref：定义基本类型 、对象的响应式数据

- reactive：定义对象 、 数组的响应式数据，解构会丢失响应式

- toRef：基于响应式对象的某个属性创建 ref

- toRefs：把响应式对象转成多个 ref 组成的普通对象，解构后仍保持响应式，适合解构 props、state


三、计算与侦听 API

- computed：定义计算属性

- watch：侦听指定数据变化

- watchEffect：自动收集依赖并执行副作用


四、组件与生命周期 API

- setup：组合式API 入口，替代 `beforeCreate/created`，接收 `props/context`，返回模板可用数据


- defineProps： 在 `<script setup>` 中声明 props，接收父组件传递数据，自动类型推导

- defineEmits：在 `<script setup>` 中声明自定义事件，子组件向父组件通信，支持类型校验

- 生命周期钩子：
  - 创建阶段：setup
  - 挂载阶段：onBeforeMount、onMounted
  - 更新阶段：onBeforeUpdate、onUpdated
  - 卸载阶段：onBeforeUnmount、onUnmounted

- defineOptions：在 `<script setup>` 中配置组件选项

- 模板 ref：获取 DOM 元素或子组件实例

五、进阶响应式 API（性能优化）


- shallowRef：浅层响应式 ref，仅监听 `.value` 引用变化，对象内部修改不触发更新

- shallowReactive：浅层响应式 reactive，仅监听第一层属性变化，嵌套属性修改不触发更新

- readonly：深层只读响应式，禁止任何层级修改，保护数据安全

- shallowReadonly：浅层只读响应式，仅禁止第一层修改，嵌套属性可修改

- toRaw：获取响应式对象的原始对象，绕过代理，修改不触发视图更新，适合性能优化 、第三方库

- markRaw：标记对象永不被代理，避免第三方库实例 、 大对象被转为响应式，提升性能

- customRef：自定义 ref 实现，手动控制 `track/trigger`，实现防抖 、持久化 、 校验等自定义逻辑


六、组件通信方式

- 父子通信：props（父→子） + emit（子→父）
- 跨级通信：provide/inject（依赖注入）
- 全局通信：Pinia /mitt（事件总线）
- 插槽通信：普通插槽 / 作用域插槽（子组件传数据给父组件渲染）
