## props


props是使用频率最高的一种通信方式，常用与 ：`父 ↔ 子`。

- 若 父传子：属性值是非函数。

- 若 子传父：属性值是函数。


::: code-group

```vue [父组件]

<template>
  <div class="father">
    <h3>父组件，</h3>
		<h4>我的车：{{ car }}</h4>
		<h4>儿子给的玩具：{{ toy }}</h4>
    <!-- 属性传递 -->
		<Child :car="car" :getToy="getToy"/>
  </div>
</template>

<script setup lang="ts" name="Father">
	import Child from './Child.vue'
	import { ref } from "vue";
	// 数据
	const car = ref('奔驰')
	const toy = ref()
	// 方法
	function getToy(value:string) {
		toy.value = value
	}
</script>


```

```vue [子组件]

<template>
  <div class="child">
    <h3>子组件</h3>
		<h4>我的玩具：{{ toy }}</h4>
		<h4>父给我的车：{{ car }}</h4>
		<button @click="getToy(toy)">玩具给父亲</button>
  </div>
</template>

<script setup lang="ts" name="Child">
	import { ref } from "vue";
	const toy = ref('奥特曼')
	
	defineProps(['car','getToy'])
</script>

```

## 自定义事件

自定义事件常用于：`子 => 父`。


 emit 是子组件向父组件传递数据 / 触发事件的核心方式，本质是 “自定义事件机制”—— 子组件通过 emit 触发一个自定义事件，父组件通过 `v-on（或 @）`监听这个事件，从而实现父子组件的通信。



Vue 中父子组件的通信规则是「单向数据流」：

- 父组件 → 子组件：通过 props 传值（子组件只能用，不能直接改）；
- 子组件 → 父组件：不能直接改父组件的数据，必须通过 emit 触发父组件的方法，由父组件自己修改数据。

emit 就是子组件 “通知” 父组件 “我有事情要告诉你 / 我要改数据” 的唯一合规方式，避免直接修改父组件数据导致的数据流混乱。


**emit 的本质：不是 “传数据”，是 “触发事件”**

- 子组件调用 `emit('xxx', 数据)` 时，只是 “发布了一个事件，并附带数据”；
- 父组件必须主动监听 `@xxx` 这个事件，才能接收到数据；
- 如果父组件没监听，子组件的 emit 不会有任何效果（也不会报错）。


::: tip

$event的值

如果是原生事件：

- 事件名是特定的（click、mosueenter等等）
- 事件对象$event: 是包含事件相关信息的对象（pageX、pageY、target、keyCode）


如果是自定义事件：

- 事件名是任意名称
- 事件对象$event: 是调用emit时所提供的数据，可以是任意类型！！！


:::

::: code-group


```vue [父]

<template>
  <div>
    <h3>父组件接收的年龄：{{ parentAge }}</h3>
    <!-- 3. 第三步：父组件监听子组件触发的自定义事件 -->
    <Child @age-change="handleAgeReceive" />
  </div>
</template>

<script setup>
import { ref } from 'vue';
import Child from './Child.vue';

const parentAge = ref('');

// 4. 处理子组件传递的数据
const handleAgeReceive = (age) => {
  parentAge.value = age; // 父组件自己修改数据，符合单向数据流
  console.log('子组件传递的年龄：', age);
};
</script>

```


```vue [子]
<template>
  <div>
    <input 
      type="number" 
      v-model="age" 
      @change="handleAgeChange"
      placeholder="输入年龄"
    >
  </div>
</template>

<script setup>
// 1. 第一步：声明要触发的事件（可选，但强烈推荐——类型校验+代码提示）
const emit = defineEmits(['age-change']); // 声明自定义事件名

// 2. 响应式数据
const age = ref('');

// 3. 触发事件的方法
const handleAgeChange = () => {
  // 2. 第二步：调用 emit 触发事件，第一个参数是事件名，后续是要传递的参数
  emit('age-change', age.value); // 向父组件传递 age 值
};
</script>

```

## mitt

mitt 是一个超轻量（≈200B）的事件总线库，专门用于实现「非父子组件」（如兄弟、跨层级、任意组件）之间的通信，也是 Vue3 中替代已移除的 `Vue.prototype.$bus`（Vue2 事件总线）的主流方案。



接收数据的： 提前绑定好事件（提前订阅消息）

提供数据的：在合适的时候触发事件（发布消息）


::: code-group

```js [创建全局mitt实例]

// 引入 mitt
import mitt from 'mitt';

// 创建 mitt 实例（全局唯一）
const emitter = mitt();

// 导出实例，供所有组件使用
export default emitter;

```

```vue [组件A]

<!-- 触发事件 / 发送消息 -->
<template>
  <button @click="sendMsg">给组件B发消息</button>
</template>

<script setup>
import emitter from '@/utils/bus';

// 触发事件：第一个参数是「事件名」，第二个是「要传递的数据」
const sendMsg = () => {
  emitter.emit('msg-change', { name: '张三', age: 18 });
  // 也可以触发无参数事件：emitter.emit('empty-event')
};
</script>



```


```vue [组件B]

<!-- 监听事件 / 接收消息 -->

<template>
  <div>接收的消息：{{ msg }}</div>
</template>

<script setup>
import { ref, onMounted, onUnmounted } from 'vue';
import emitter from '@/utils/bus';

const msg = ref({});

// 监听事件：第一个参数「事件名」，第二个「回调函数（接收数据）」
const handleMsg = (data) => {
  msg.value = data;
  console.log('收到消息：', data); // { name: '张三', age: 18 }
};

// 组件挂载时监听事件（避免提前监听导致内存泄漏）
onMounted(() => {
  emitter.on('msg-change', handleMsg);
  // 监听所有事件（调试用）：
  // emitter.on('*', (type, data) => console.log(type, data))
});

// 组件卸载时移除监听（核心！防止内存泄漏）
onUnmounted(() => {
  emitter.off('msg-change', handleMsg);
  // 移除所有监听：emitter.all.clear()
});
</script>

```

## v-model


`v-model` 是 Vue 提供的语法糖，核心作用是实现表单元素（输入框、单选框、下拉框等）和组件数据的 “双向绑定”
 —— 既让数据变化自动同步到表单视图，也让表单输入自动同步到数据，无需手动写 `@input + :value `这类重复代码


`v-model` 本质是「:value + 对应事件」的语法糖，不同表单元素的事件略有差异：

![LOGO](/public/image/base/ScreenShot_2026-03-21_131356_717.png)



`v-model` 作用：简化双向绑定代码，统一表单 / 组件的双向绑定逻辑；

- 原生表单元素：value + 事件 语法糖；

- 自定义组件：modelValue prop + update:modelValue emit 语法糖。


`v-model` 原理

本质是「单向数据流 + 事件通知」的封装（并非真正的 “双向绑定”，而是 Vue 帮你做了 “数据→视图”+“视图→数据” 两步操作）；

核心遵循 Vue “单向数据流” 原则：数据只能由父组件传给子组件，子组件通过事件通知父组件修改数据，而非直接修改props。



::: code-group

```vue [原生表单]

<template>
  <!-- ✅ 语法糖写法 -->
  <input v-model="msg" />

  <!-- ❗️ 等价的底层实现（手动绑定） -->
  <input :value="msg" @input="msg = $event.target.value" />

  <p>数据：{{ msg }}</p>
</template>

<script setup>
import { ref } from 'vue';
const msg = ref('');
</script>

<!-- 

1、初始化时，:value="msg" 把数据 msg 同步到输入框视图

2、用户输入时，触发 input 事件，通过 @input 把输入框的最新值（$event.target.value）赋值给 msg，
实现视图→数据的同步；

3、msg 变化后，Vue 的响应式系统又会更新 :value，实现数据→视图的同步。
-->



```


```vue [自定义组件]

<!-- 
组件的 v-model 做了重构，更灵活，核心原理是「props + emit」的语法糖：

默认规则：父组件给子组件传 modelValue prop，
子组件触发 update:modelValue 事件，父组件自动更新数据。 

-->

<!-- 子组件 -->
<template>
  <!-- 子组件内部绑定原生输入框 -->
  <input 
    :value="modelValue" 
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>

<script setup>
// 接收父组件传递的 modelValue prop
const props = defineProps(['modelValue']);
// 声明要触发的更新事件（可选，推荐加）
const emit = defineEmits(['update:modelValue']);
</script>


<!-- 父组件 -->
<template>
  <!-- ✅ 自定义组件用 v-model -->
  <MyInput v-model="parentMsg" />
  <p>父组件数据：{{ parentMsg }}</p>

  <!-- ❗️ 等价的底层实现（props + emit） -->
  <MyInput 
    :modelValue="parentMsg" 
    @update:modelValue="parentMsg = $event"
  />
</template>

<script setup>
import { ref } from 'vue';
import MyInput from './MyInput.vue';
const parentMsg = ref('');
</script>

<!-- 

父组件的 v-model="parentMsg" 等价于给子组件传 
:modelValue="parentMsg" + 监听 @update:modelValue；

子组件通过 props 接收 modelValue，绑定到原生输入框的 value；

子组件输入时触发 input 事件，通过 $emit('update:modelValue', 新值) 通知父组件；

父组件监听到事件后，自动把新值赋值给 parentMsg，完成双向绑定。

 -->


```


```vue [自定义修饰符]

<!-- 父组件：加自定义修饰符 -->
<MyInput v-model.capitalize="msg" />

<!-- 子组件：接收修饰符 -->
<script setup>
const props = defineProps({
  modelValue: String,
  modelModifiers: { default: () => ({}) } // 接收修饰符
});

// 处理修饰符：如果有 capitalize，把输入值转大写
const handleInput = (e) => {
  let value = e.target.value;
  if (props.modelModifiers.capitalize) {
    value = value.toUpperCase();
  }
  emit('update:modelValue', value);
};
</script>

```

## $attrs


`$attrs` 用于实现当前组件的父组件，向当前组件的子组件通信 `（祖→孙）`。


具体说明：$attrs 是一个对象，包含所有父组件传入的标签属性。


注意：$attrs 会自动排除 props 中声明的属性(可以认为声明过的 props 被子组件自己“消费”了)


::: code-group


```vue [父]

<template>
  <div class="father">
    <h3>父组件</h3>
    <!-- v-bind 后面可以跟一个对象 -->
		<Child :a="a" :b="b" :c="c" :d="d" 
          v-bind="{x:100,y:200}" 
          :updateA="updateA"/>
  </div>
</template>

<script setup lang="ts" name="Father">
	import Child from './Child.vue'
	import { ref } from "vue";
	let a = ref(1)
	let b = ref(2)
	let c = ref(3)
	let d = ref(4)

	function updateA(value){
		a.value = value
	}
</script>



```


```vue [子]

<template>
	<div class="child">
		<h3>子组件</h3>
    <!-- v-bind 后面可以跟一个对象 -->
		<GrandChild v-bind="$attrs"/>
	</div>
</template>

<script setup lang="ts" name="Child">
	import GrandChild from './GrandChild.vue'
</script>


```


```vue [孙]

<template>
	<div class="grand-child">
		<h3>孙组件</h3>
		<h4>a：{{ a }}</h4>
		<h4>b：{{ b }}</h4>
		<h4>c：{{ c }}</h4>
		<h4>d：{{ d }}</h4>
		<h4>x：{{ x }}</h4>
		<h4>y：{{ y }}</h4>
		<button @click="updateA(666)">点我更新A</button>
	</div>
</template>

<script setup lang="ts" name="GrandChild">
	defineProps(['a','b','c','d','x','y', 'updateA'])
</script>


```

## provide 和 inject

实现祖孙组件直接通信

在祖先组件中通过 `provide` 配置向后代组件提供数据

在后代组件中通过`inject`配置来声明接收数据



::: code-group


```vue [父]
<template>
  <div class="father">
    <h3>父组件</h3>
    <h4>资产：{{ money }}</h4>
    <h4>汽车：{{ car }}</h4>
    <button @click="money += 1">资产+1</button>
    <button @click="car.price += 1">汽车价格+1</button>
    <Child/>
  </div>
</template>

<script setup lang="ts" name="Father">
  import Child from './Child.vue'
  import { ref, reactive, provide } from "vue";
  // 数据
  let money = ref(100)
  let car = reactive({
    brand:'奔驰',
    price:100
  })
  // 用于更新money的方法
  function updateMoney(value:number){
    money.value += value
  }
  // 提供数据
  provide('moneyContext',{ money, updateMoney })
  provide('car',car)
</script>


```

```vue [子]


<template>
  <div class="grand-child">
    <h3>我是孙组件</h3>
    <h4>资产：{{ money }}</h4>
    <h4>汽车：{{ car }}</h4>
    <button @click="updateMoney(6)">点我</button>
  </div>
</template>

<script setup lang="ts" name="GrandChild">
  import { inject } from 'vue';
  // 注入数据
  let { money, updateMoney } = inject('moneyContext', {
    money: 0,
    updateMoney:(x:number) => {}
  })
  let car = inject('car')
</script>
```

## 作用域插槽

作用域插槽是什么：带数据传递能力的插槽，子组件传数据给父组件，父组件自定义渲染；

作用域插槽设计的初衷，就是让子组件把自己内部的、父组件拿不到的数据 “暴露” 给父组件，让父组件能基于这份数据自定义渲染样式。


子组件：`<slot :自定义属性="子组件数据">` 暴露数据；

父组件：`<template #default="{ 解构属性 }"> `接收数据并渲染；

原理：插槽 props 的双向传递（子传数据 → 父自定义渲染 → 内容回插子组件）；

简单来说，作用域插槽就是 “子组件给父组件递数据，父组件按自己的想法渲染，再塞回子组件里”


::: code-group

```vue [子组件]
<!-- 
提供数据 + 绑定插槽属性
 -->

<template>
  <div class="list-wrapper">
    <h3>用户列表（子组件提供数据）</h3>
    <!-- 1. 定义作用域插槽：通过 v-bind 把数据传给父组件 -->
    <slot 
      v-for="user in userList" 
      :key="user.id"
      :user="user"  <!-- 绑定子组件数据：user（自定义属性名） -->
      :index="index" <!-- 可绑定多个属性 -->
    ></slot>
  </div>
</template>

<script setup>
import { ref } from 'vue';

// 子组件的内部数据（父组件需要用这个数据渲染）
const userList = ref([
  { id: 1, name: '张三', age: 18 },
  { id: 2, name: '李四', age: 20 },
  { id: 3, name: '王五', age: 22 }
]);
</script>

<!-- 
<slot> 标签上通过 v-bind:xxx="数据" 
把子组件的内部数据 “暴露” 给父组件，
这些绑定的属性称为「插槽 props」。
 -->

```

```vue [父组件]

<!-- 
接收插槽 props + 自定义渲染
 -->

<template>
  <div>
    <!-- 2. 父组件使用子组件，通过 slot-scope 接收插槽 props（Vue3 简写为 #default="slotProps"） -->
    <UserList>
      <!-- 方式 1：完整写法（v-slot:default="插槽接收的参数"） -->
      <template v-slot:default="slotProps">
        <div class="item">
          索引：{{ slotProps.index }} →
          姓名：{{ slotProps.user.name }}，
          年龄：{{ slotProps.user.age }}
        </div>
      </template>

      <!-- 方式 2：简写（#default，解构插槽 props，推荐） -->
      <!-- <template #default="{ user, index }">
        <div class="item">
          索引：{{ index }} → 姓名：{{ user.name }}，年龄：{{ user.age }}
        </div>
      </template> -->
    </UserList>
  </div>
</template>

<script setup>
import UserList from './UserList.vue';
</script>

```


:::


作用域插槽的本质是「插槽 props 的双向传递」，底层逻辑：

- 子组件暴露数据：子组件在 `<slot>` 上绑定的属性（如 :user="user"），会被 Vue 收集为「插槽 props 对象」；

- 父组件接收数据：父组件通过 `v-slot:xxx="slotProps"` 接收这个 props 对象，相当于子组件把数据 “传给” 了父组件；

- 父组件渲染后回传：父组件用接收到的数据自定义渲染内容，渲染后的内容会被插入到子组件的 `<slot>` 位置，完成整个流程。
