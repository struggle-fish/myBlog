## 大 O 表示法

大O表示法 = 当数据量爆炸级增长时，真正决定程序会不会超时的那个部分，用最简单的式子来表达它。


![LOGO](/public/image/datastructure/ScreenShot_2026-02-16_091526_560.png)


在算法分析里，我们只关心 **「当数据量超级大时，哪个部分最耗时间」**

其他小部分（常数、加的几行代码、低阶项）到最后几乎可以完全忽略。

用大O表示法，就是把这个「只关心最耗时的部分」这件事，用一个数学记号表达出来，把那些最后变得微不足道的部分全部扔掉。

- `T₁(n) = 3n + 3 ≈ 3n` 多加了个3，后面根本不影响,只看 n
- `T₂(n) = n² + 3n + 1000 ≈ n²`,`n²` 已经远远大于 3n 和 1000 了, 只看 n²
- `T₃(n) = n³ + n² + 99999999 ≈ n³` ,n³ 完全碾压 n² 和常数项,只看 n³
- `T(n) = O(f(n))`,这个算法「最坏/平均」情况下，增长速度和 f(n) 同一个量级,`≈ f(n)`


## 大O表示法最核心的几句话

- `O( )` 里面只写最高阶项，常数、低阶项全部不要
- 不写系数 `O(3n) → O(n)`  `O(7n² + 200n + 5) → O(n²)`，n 足够大时，只看最高次项，系数也直接看成 1
- 它描述的是「增长速度的量级」，而不是精确时间
- 常见的量级从快到慢（越靠右越慢）：

`O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)`

**n 足够大时，只看最高次项，系数也直接看成 1。**

## 加法规则和乘法规则


### 加法规则


![LOGO](/public/image/datastructure/ScreenShot_2026-02-16_094547_396.png)

1. 当 n 变得非常大的时候，到底谁才是真正“吃时间”的部分？
2. 多项式相加 vs 相乘，处理方式完全不同


**为什么可以“忽略”一些项**

当 n 非常大的时候（比如 n = 10⁶、10⁸、10¹⁰ ……）

以下这些项的“重要性”差距极大：


![LOGO](/public/image/datastructure/ScreenShot_2026-02-16_095137_412.png)


你会发现：

- n³ 已经比 n² 大了上百万倍、十亿倍
- n² 已经把 n 甩开很远
- 常数、999999、甚至 10⁶ 这种多项式低阶的，在 n=10⁹ 时几乎可以直接忽略

所以才有了大家最熟悉的那句话：`n 足够大时，只看最高次项，系数也直接看成 1`

- `T₁(n) = 3n + 3 → O(n)`
- `T₂(n) = n² + 3n + 1000 → O(n²)`
- `T₃(n) = n³ + n² + 9999999 → O(n³)`

这就是“加法规则”的本质。

加法规则（多项式相加时）`T(n) = T₁(n) + T₂(n) → O(max( f(n), g(n) ))` 写得更直白就是：**只保留最高次的项，前面系数直接忽略（当成 1）** 
比如 `n³ + n² + 100n + 99999 + log n + 7 最终写成：O(n³)`


### 乘法规则

**乘法规则（多项式相乘时）** 相乘的时候不能只保留最高项，而是要把每一项都保留下来相乘

假如  `T(n) = (n³ + n² log n) × n²`

应该展开成：

- `= n³ × n² + n² log n × n²`
- `= n⁵ + n⁴ log n`

所以最终复杂度是： `O(n⁵ + n⁴ log n)`


- 先把括号里的每一项都分别去乘后面的项
- 得到一堆新项
- 再在新的一堆项里，找出最高次的那个（或前几个比较接近的）


### 快速记忆口诀

- 加的时候 → 只留老大（最高次），别人全部丢掉
`n³ + 999n² + 10¹⁰ → O(n³)`

- 乘的时候 → 每一项都要去乘，乘完再找老大
`(n³ + n²) × (n² + log n) → n⁵ + n³ log n + n⁴ + n² log n → O(n⁵)`



## 如何理解 O(n log n) 和 O(log n)


![LOGO](/public/image/datastructure/ScreenShot_2026-02-16_092756_694.png)


:::tip
- O(log n) → “每次砍一半”，超级快
- O(n log n) → “先扫一遍 + 每次砍一半”，比 O(n) 慢大概 30 倍左右，但远比 O(n²) 快
:::


**生活中的例子理解log n**

想象你在 `1亿人 `的电话本里找一个人：

- O(n)：从头翻到尾 → 最坏要翻 1亿次
- O(log n)：用二分查找（每次翻到中间，问“这个人名字在左边还是右边？”）
→ 最多只需 `27` 次 就能找到`（因为 2²⁷ ≈ 1.34亿）`

## 对数入门

多少个既定的数相乘会等于另一个数？


:::tip

例子：多少个 2 相乘会等于 8？

答案：2 × 2 × 2 = 8，所以需要把 3 个 2 相乘来得到 8

所以对数是 3

:::

我们这样写"3个2相乘的积为8"：

**`log2(8) = 3`** (需要多少个 2 相乘，来得到 8)


![LOGO](/public/image/datastructure/logarithm-concept.svg)


::: tip

相乘的数叫 "底"，而对数的符号是 "log"，所以我们可以说：

"8的以2为底的对数是 3"

或 "8的log（以2为底）是 3"


:::



请留意，这牵涉到三个数：
- 底：相乘的数（在以上的例子的 "2"）
- 多少个底相乘（3个，这就是 对数）
- 相乘起来的积（在以上的例子的 "8"）



例子： log5(625) 是多少？

`问题是 "需要多少个 5 相乘，来得到 625？"`

5 × 5 × 5 × 5 = 625，所以需要4 个 5

`答案：log5(625) = 4`
