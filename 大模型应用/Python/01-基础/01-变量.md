# 变量



## 什么是变量

变量就是可以变化的量，量指的是事物的状态，比如人的年龄、性别，游戏角色的等级、金钱等等

变量是对对象的引用（Reference）

```py

a = 100

```

- Python 先在内存里创建一个 整数对象，它的值是 100。
- 然后把变量名 a 这个标签贴到这个 100 对象上。
- a 本身不存 100，它只存了这个对象的内存地址（引用）。

```text

内存仓库：
   ┌─────────────────┐
   │  整数对象: 100   │  ← 这里真正存着 100
   └────────▲────────┘
            │
           "a"   ← 变量 a 就是这个标签，指向上面的对象

```

## 变量的基本使用

Python 定义变量不用提前声明类型，直接赋值就行，先定义后使用


```py

name = '小铜钱' # 变量定义

print(name) # 变量的引用


# 基本赋值
name = "张三"          # 字符串
age = 18               # 整数
height = 1.75          # 浮点数
is_student = True      # 布尔值
hobbies = ["打球", "唱歌"]   # 列表

print(name, age, height, is_student)

```


## 变量类型和内存地址

- id() : 反应的是变量的内存地址，内存地址不同id则不同

- type(): 不同类型的值用来表示记录不同的状态

```py

name = "小明"
age = 18
scores = [90, 85, 95]

print(type(name))     # <class 'str'>
print(type(age))      # <class 'int'>
print(type(scores))   # <class 'list'>

print(id(name))       # 查看变量指向的对象的内存地址

```

## is 和 ==

`==`：比较值是否相等（value equality）

`is`：比较是否是同一个对象（identity / 内存地址是否相同），`a is b 等价于：id(a) == id(b)`


- 判断是否为 None：永远用 is None 或 is not None

- 判断布尔、空值：可以用 `is True / is False`，但更推荐直接 `if data`:（利用 Python 的真值判断）

- 其他所有情况（数字、字符串、列表、字典等）：几乎都用 `==`

- 需要检查是否为同一个对象时，才用 `is`（例如单例模式、缓存对象等）

python 语法中没有常量的概念，但是在程序的开发过程中会涉及到常量的概念

小写字母全为大写代表常量，这只是一种约定、规范

:::




## 垃圾回收

当内存里的某个对象，没有任何变量指向它了，Python 就自动把它删掉、释放内存 —— 这就叫垃圾回收。

**没人用的对象 = 垃圾**

**Python 自动清垃圾 = 垃圾回收**

Python 的垃圾回收（Garbage Collection，简称 GC）机制 是 Python 自动内存管理的重要部分。它让程序员不用手动 `malloc / free`，大大减少了内存泄漏的风险。


Python 的 GC 不是单一机制，而是三种机制协同工作：

- 引用计数（Reference Counting） —— 主要机制，实时回收

- 标记-清除（Mark-Sweep） —— 解决循环引用问题

- 分代回收（Generational GC） —— 提高效率，以空间换时间


📢：`del` 是解除变量名x 与值10的绑定关系，不是删除对象

### 引用计数

每个 Python 对象内部都有一个计数器（叫 ob_refcnt），记录有多少个引用指向它。

- 新建一个对象，引用计数 = 1
- 每多一个变量指向它，计数 +1
- 删除一个引用（变量重新赋值、del、离开作用域等），计数 -1
- 当计数 变为 0 时，对象立刻被回收，内存立即释放


```py

import sys

a = [1, 2, 3]          # 创建列表对象，引用计数 = 1
b = a                  # 又指向它，计数 = 2
c = a                  # 计数 = 3

print(sys.getrefcount(a))  # 输出通常是 4（因为 getrefcount 本身也算一个临时引用）

del a                  # 计数 -1 → 2
b = None               # 计数 -1 → 1
c = None               # 计数 -1 → 0 → 对象立刻被回收！

```

优点：简单、实时（对象一没用就回收）

缺点：无法处理循环引用

### 循环引用问题（引用计数的致命弱点）

两个或多个对象互相引用，导致大家计数都不为 0，但外部已经没人用了。

```py
class Node:
    def __init__(self):
        self.next = None

a = Node()
b = Node()

a.next = b     # a 引用 b
b.next = a     # b 引用 a

del a
del b

```

- a 和 b 互相引用，引用计数都是 1
- 但程序里已经没有变量能访问它们了（不可达）
- 单纯靠引用计数永远不会回收它们 → 内存泄漏！

循环引用：

直接引用指的是从栈区出发直接引用到的内存地址，间接引用指的是从栈区出发引用到堆区后再进一步引用到的内存地址

![LOGO](/public/image/python/ScreenShot_2026-04-11_105531_246.png)

同时删除l1与l2时，会清理到栈区中l1与l2的内容，此时没有直接引用了，外界永远访问不到了
但是堆内存里还存在相互引用，本该回收的，引用计数不为0 ，此时引用计数就失效了

### 标记-清除（Mark-Sweep）

Python 在容器对象（list、dict、set、自定义类实例、tuple 等能包含其他对象的类型）上额外维护一个循环垃圾收集器。

工作流程（简化版）：

- 把所有可能参与循环引用的容器对象放到一个“待检查”集合里
- 标记阶段：从根对象（全局变量、栈上的局部变量等）出发，遍历所有可达对象，打上“活着”的标记
- 清除阶段：把没被标记的对象当作垃圾回收掉

这个过程会暂停程序（Stop-The-World），但只针对容器对象，效率还算可接受。

::: tip

1、标记

通俗地讲就是：标记的过程就相当于从栈区出发一条线，“连接”到堆区，再由堆区间接“连接”到其他地址，凡是被这条自栈区起始的线连接到内存空间都属于可以访达的，会被标记为存活

具体地：标记的过程其实就是，遍历所有的GC Roots对象(栈区中的所有内容或者线程都可以作为GC Roots对象），然后将所有GC Roots的对象可以直接或间接访问到的对象标记为存活的对象，其余的均为非存活对象，应该被清除。 注意是关注的是栈区
                          
2、清除
清除的过程将遍历堆中所有的对象，将没有标记存活的对象全部清除掉。

:::



### 分代回收（Generational Collection）

Python 发现：大多数对象“朝生夕死”（创建后很快就不用了），少数对象会活得很久。

所以把对象分成 3 代（Generation）：

- 第 0 代（年轻代）：新创建的对象都在这里
- 第 1 代：从第 0 代活下来的对象
- 第 2 代（老年代）：从第 1 代活下来的对象（活得最久）



回收策略（以空间换时间）：

- 新对象多 → 频繁回收第 0 代（快，因为对象少且大部分是垃圾）
- 第 0 代回收一定次数后，才回收第 1 代
- 第 1 代回收一定次数后，才回收第 2 代（老年代回收最少）

默认阈值大致是：

- 第 0 代：分配 700 个新对象左右就触发一次
- 第 1 代：第 0 代回收 10 次后触发
- 第 2 代：第 1 代回收 10 次后触发

这样大部分垃圾在年轻代就被快速回收了，老对象很少被打扰。

::: tip

- 引用计数 是主力，实时回收大多数对象。
- 标记-清除 是辅助，专门处理循环引用（只针对容器对象）。
- 分代回收 是优化，把对象按“年龄”分代，年轻对象频繁回收，老对象很少动，提高整体效率。

:::
