## 概念
 
什么是继承？

继承是将多个类的相同属性和行为抽取到单独一个类中，那么多个类无需在定义这些共性属性和行为
只要继承这个单独类即可继承这些属性和行为了。其中多个类成为子类（派生类），单独的这个类成为父类（基类，超类）

- 继承是面向对象三大特征之一(封装,**继承**,多态)
- 可以使得子类使用父类的公共属性和公共方法,还可以再子类中重新定义,追加属性和方法
- 只允许单继承，不允许多继承（一个儿子只能有一个爹，不能有多个），但是可以多层继承（一个儿子可以有一个爹，这个爹可以有一个自己的爹，以此类推）
- 好处是提供了代码的复用性,维护性
- 让类和类之间产生了关系,是多态的前提

![LOGO](/public/image/javapublic/ScreenShot_2026-01-27_151151_511.png)

```java
// 格式
public class 子类名 extends 父类名 {
    // 子类内容
}

public class Fu { // 父类,基类,超类
    public Fu() {
       System.out.println("父类构造方法");
    }
    
    // 方法
    public void eat() {
       System.out.println("父类方法");
    }
}

// 子类,派生类
public class Zi extends Fu { // 继承
    public Zi() {
       // 默认调用父类无参构造方法
       super(); // 调用父类构造方法,只能出现在子类构造方法的第一行
       System.out.println("子类构造方法");
    }
    // 追加子类方法
    public void sleep() {
       System.out.println("子类方法");
    }
}

```




## 语法特点

- 只支持单继承，不支持多继承，但是可以多层继承
- 在子类中访问一个变量的规则：就近原则，先在子类局部范围找，然后子类成员范围找，然后父类成员范围找，
如果找不到则报错，但是父类中私有成员，子类不能访问

## 方法访问特点

通过子类对象访问一个方法：就近原则
- 先在子类局部范围找，有则使用
- 然后在子类成员范围找，有则使用
- 然后在父类成员范围找，有则使用
- 如果都没有，则报错（不能访问父类私有成员）
