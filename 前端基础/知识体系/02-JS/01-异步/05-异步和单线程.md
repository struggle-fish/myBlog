# 异步和单线程


## 同步和异步的区别是什么

**同步（Synchronous）**

- 代码按顺序执行
- 前一个任务没完成，后面全部阻塞等待
- JS 主线程会卡住
- 例如：for循环、alert、同步 AJAX、普通代码


**异步（Asynchronous）**

- 代码不会等待，立即继续执行后面
- 异步任务交给浏览器 / Node 异步模块处理
- 完成后通过回调 / Promise /await通知结果
- 例如：setTimeout、ajax/fetch、事件监听、文件读取、定时器


## 单线程和异步

JS 为什么必须要有异步？

只有一个主线程，只能同时做一件事儿，如果全是同步，一遇到网络请求、定时器、文件读取，页面就卡死不动。

所以：DOM 事件、AJAX、定时器、fetch、axios、Promise、async/await 全都是异步。

JS和DOM渲染共用同一个线程，因为JS可以修改DOM结构


异步是基于回调函数的

::: code-group

```js [异步]

console.log('开始');
setTimeout(() => {
  console.log('我是异步，最后执行');
}, 1000);
console.log('结束');
```

```js [同步]

console.log('开始');
alert('我是同步弹窗，必须点掉才能继续'); // 阻塞！
console.log('结束');

```

:::



## 手写用Promise加载一张图片

```js

function loadImage(url) {
  return new Promise((resolve, reject) => {
    // 1. 创建图片对象
    const img = new Image();

    // 2. 图片加载成功
    img.onload = function () {
      resolve(img); // 返回图片DOM
    };

    // 3. 图片加载失败
    img.onerror = function () {
      reject(new Error('图片加载失败：' + url));
    };

    // 4. 开始加载
    img.src = url;
  });
}



// 调用
async function showImage() {
  try {
    const img = await loadImage('https://example.com/1.jpg');
    document.body.appendChild(img);
  } catch (err) {
    console.log(err);
  }
}

showImage();
```
