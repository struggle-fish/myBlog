# 事件


:::tip

- 事件绑定：给元素添加事件监听器（推荐 addEventListener）
- 事件冒泡：事件从目标元素向上传播到祖先元素
- 事件代理：利用冒泡机制，把事件绑定在父元素上，通过 e.target 处理子元素事件（性能更好、支持动态元素）

:::

## 事件绑定

定义：给 DOM 元素绑定事件监听器，让元素在特定事件发生时执行对应函数。

### addEventListener

addEventListener（现代推荐写法），可以绑定多个处理函数、支持事件捕获/冒泡阶段、可以移除事件

```js

const btn = document.querySelector('#btn')

// 绑定点击事件
btn.addEventListener('click', function(e) {
  console.log('按钮被点击了')
})

// 支持多个相同事件
btn.addEventListener('click', handler1)
btn.addEventListener('click', handler2)

// 移除事件（必须使用同一个函数引用）
btn.removeEventListener('click', handler1)

```

### onclick 等属性绑定

不推荐大量使用

```js
btn.onclick = function() {
  console.log('点击了')
}
// 注意：这种方式只能绑定一个处理函数，后绑定的会覆盖前面的

```


编写一个通用的事件监听函数

```js

function bindEvent(elem, type, fn) {
  elem.addEventListener(type, fn)
}

// 调用
const a = document.getElementById('link1')

bindEvent(a, 'click', e => {
  e.preventDefault()
  alert('click')
})

```



## 事件冒泡

定义：当一个元素触发某个事件时，这个事件会从当前元素开始，一层层向上传播到它的父元素、祖先元素，直到 document 或 window。

这是 DOM 事件机制中非常重要的概念，几乎所有事件（click、mousedown、keydown 等）默认都会冒泡。

### 事件冒泡流程


事件流三个阶段

当你点击一个元素时，事件会按照以下顺序传播（事件流）：


1. 捕获阶段（Capturing Phase） —— 从外到内

事件从最外层（window）开始，逐层向下传播到目标元素。


2. 目标阶段（Target Phase）

事件到达真正被点击的目标元素，并在该元素上触发。


3. 冒泡阶段（Bubbling Phase） —— 从内到外（最重要）

事件从目标元素开始，逐层向上冒泡到它的父元素、祖父元素……直到 window。


```html
<div id="grandpa">
  <div id="parent">
    <button id="child">点击我</button>
  </div>
</div>

```

```js

document.getElementById('grandpa').addEventListener('click', () => {
```

### 重要API

- `e.stopPropagation() ` ： 阻止事件继续冒泡（常用）-> 阻止向上传播

- `e.stopImmediatePropagation()` ： 阻止当前元素其他事件处理函数执行 + 阻止冒泡  -> 更强力阻止

- `addEventListener(..., true)` ： 在捕获阶段触发  -> 第三个参数设为 true

- `event.target` ： 真正触发事件的元素（最常用） -> 事件代理核心

- `event.currentTarget`： 当前正在处理事件的元素

**事件冒泡就是事件从目标元素开始，沿着 DOM 树逐层向上传播的过程，这是浏览器默认的行为。开发者可以通过 stopPropagation() 阻止冒泡，通过事件代理利用冒泡机制实现高效的事件处理。**



## 事件代理


定义：利用事件冒泡机制，把原本应该绑定在多个子元素上的事件，统一绑定在它们的父元素或祖先元素上，通过事件对象的 e.target 来判断点击的是哪个子元素。


为什么需要事件代理？


传统写法的问题（给 1000 个 li 分别绑定事件）：

```js

// 性能差 + 内存占用高 + 新增元素无效
const lis = document.querySelectorAll('li')
lis.forEach(li => {
  li.addEventListener('click', () => {
    console.log(li.textContent)
  })
})

```

使用事件代理的正确写法：

```js

const ul = document.querySelector('ul')

ul.addEventListener('click', (e) => {
  // e.target 是实际被点击的元素
  if (e.target.tagName === 'LI') {
    console.log('点击了：', e.target.textContent)
  }
})

```


事件代理的核心优势：

- 性能更好， 只绑定一个事件监听器
- 内存占用低，避免给大量元素绑定事件
- 支持动态元素，后续动态添加的 li 也能自动响应事件
- 代码更简洁 ，逻辑集中管理


## 事件流的完整过程

事件从触发到结束会经历三个阶段：

1. 捕获阶段（Capturing Phase）—— 从外向内（window → document → 目标元素）

2. 目标阶段（Target Phase）—— 到达真正触发事件的元素

3. 冒泡阶段（Bubbling Phase）—— 从内向外（目标元素 → 父元素 → ... → window）


addEventListener 的第三个参数：

```js

element.addEventListener('click', handler, useCapture)

// useCapture = false（默认）→ 冒泡阶段触发
// useCapture = true       → 捕获阶段触发

```


## 开发实践

```js

// 推荐的现代事件代理写法
function delegate(parent, selector, eventType, handler) {
  parent.addEventListener(eventType, function(e) {
    const target = e.target.closest(selector)   // 向上查找最近的匹配元素
    if (target && parent.contains(target)) {
      handler.call(target, e)
    }
  })
}

// 使用示例
delegate(document.body, '.btn', 'click', function(e) {
  console.log('点击了动态按钮', this)
})

```
