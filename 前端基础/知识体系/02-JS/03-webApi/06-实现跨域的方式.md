# 跨域的方式


## 什么是跨域

跨域：指浏览器从一个源（Origin） 去请求另一个源的资源时，受到浏览器同源策略的限制。


什么是“源”（Origin）？

一个 URL 的源由以下三部分组成：

- 协议（protocol）：http 或 https
- 域名（host）：example.com
- 端口（port）：默认 80（http）或 443（https）

只有当协议 + 域名 + 端口 完全相同时，才算同源。否则就是跨域。



## 为什么会有跨域？（同源策略）

浏览器实行同源策略（Same-Origin Policy），主要目的是安全：

防止恶意网站通过 JS 读取其他网站的 Cookie、LocalStorage 等敏感信息

防止恶意网站冒充用户身份，向其他网站发起恶意请求（CSRF 攻击）

如果没有同源策略，用户登录银行网站后，打开一个恶意网站，该恶意网站就可以偷偷读取银行网站的 Cookie 并发起转账操作。

## CORS

CORS 是 W3C 标准，目前最主流、最推荐的跨域解决方案。

工作原理：

浏览器在发起跨域请求时，会自动携带 Origin 头字段。服务器通过响应头告诉浏览器是否允许该源访问。


关键响应头：

```http

Access-Control-Allow-Origin: https://example.com     # 允许的源（* 表示所有）
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Credentials: true               # 是否允许携带 Cookie

```

- 简单请求：`GET、POST（Content-Type 为 application/x-www-form-urlencoded、multipart/form-data、text/plain）`等，直接发起。

- 复杂请求：会先发一个 OPTIONS 预检请求，确认服务器是否允许，再发正式请求。


后端node 示例：

```js

app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', 'https://frontend.com');
  res.header('Access-Control-Allow-Credentials', 'true');
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE');
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');

  if (req.method === 'OPTIONS') {
    res.sendStatus(200);
  } else {
    next();
  }
});

```


## 代理 proxy

在开发阶段（Webpack、Vite、Create React App 等）使用代理服务器转发请求。

```js

// vite.config.js
export default {
  server: {
    proxy: {
      '/api': {
        target: 'https://backend.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
}

```

原理：开发服务器帮你转发请求，绕过浏览器同源限制。


## JSONP

原理：利用 `<script>` 标签没有同源限制的特性。


```html

<script src="https://api.example.com/data?callback=handleData"></script>

```


```js

function jsonp({ url, params, callback }) {
  return new Promise((resolve, reject) => {
    const script = document.createElement('script')
    const callbackName = 'jsonp_' + Date.now()

    window[callbackName] = function(data) {
      resolve(data)
      document.body.removeChild(script)
      delete window[callbackName]
    }

    params = { ...params, callback: callbackName }
    const query = new URLSearchParams(params).toString()
    script.src = `${url}?${query}`
    document.body.appendChild(script)
  })
}

// 使用
jsonp({
  url: 'https://api.example.com/user',
  params: { id: 123 }
}).then(data => console.log(data))

```

缺点：只能 GET 请求、不安全、已基本被淘汰。



## postMessage

适用场景：主要用于 iframe 跨域通信、不同标签页（window）之间的通信。


`postMessage()` 是 HTML5 提供的 API，允许不同源的窗口（包括 iframe）安全地进行跨域消息传递。它不受同源策略限制。


实现方式:


```html

<iframe id="bFrame" src="https://b.com/page.html"></iframe>

<script>
// 向 iframe 发送消息
const iframe = document.getElementById('bFrame')
iframe.contentWindow.postMessage(
  { type: 'login', token: 'abc123' }, 
  'https://b.com'        // 必须指定目标源，安全考虑
)

// 接收 B 页面发来的消息
window.addEventListener('message', (event) => {
  // 必须验证来源，防止恶意网站攻击
  if (event.origin !== 'https://b.com') return
  
  console.log('收到 B 页面的消息：', event.data)
})
</script>

```

B 页面（接收方） —— 例如 `https://b.com`


```html

<script>
window.addEventListener('message', (event) => {
  if (event.origin !== 'https://a.com') return
  
  console.log('收到 A 页面的消息：', event.data)
  
  // 回复消息
  event.source.postMessage(
    { type: 'reply', status: 'success' }, 
    event.origin
  )
})
</script>

```


关键点：

- 必须验证 event.origin，防止安全风险
- event.source 可以用于回复消息

优点：不需要后端配合

缺点：只能用于窗口/iframe 通信，不适合普通前后端 API 调用



## WebSocket


适用场景：需要实时双向通信的场景（如聊天、股票行情、在线协作等）


原理： `WebSocket` 是一种独立的协议（`ws:// 或 wss://`），建立连接后，浏览器和服务器可以全双工通信，不受同源策略限制。


前端代码：


```js

const socket = new WebSocket('wss://api.example.com/chat')

// 连接成功
socket.onopen = () => {
  console.log('WebSocket 已连接')
  socket.send(JSON.stringify({ type: 'login', token: 'xxx' }))
}

// 接收消息
socket.onmessage = (event) => {
  const data = JSON.parse(event.data)
  console.log('收到服务器消息：', data)
}

// 连接关闭
socket.onclose = () => {
  console.log('连接已关闭')
}

// 错误处理
socket.onerror = (error) => {
  console.error('WebSocket 错误', error)
}


```

后端（`Node.js` 示例）：

```js

const WebSocket = require('ws')
const wss = new WebSocket.Server({ port: 8080 })

wss.on('connection', (ws) => {
  console.log('客户端已连接')
  
  ws.on('message', (message) => {
    console.log('收到客户端消息：', message)
    // 广播给所有客户端
    wss.clients.forEach(client => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message)
      }
    })
  })
})


```

优点：

- 支持全双工实时通信
- 天然支持跨域
- 性能高（头部压缩）

缺点：

- 需要后端专门支持 WebSocket 协议
- 连接管理较复杂（心跳、重连等）



## Nginx反向代理


适用场景：前后端分离项目，最推荐的跨域解决方案之一。



通过 Nginx 服务器作为中间代理，把前端请求转发到后端服务器。前端请求的是同域地址，Nginx 在服务器端完成跨域请求，从而绕过浏览器同源策略。

Nginx 配置示例

```nginx

server {
    listen       80;
    server_name  frontend.com;

    # 前端静态资源
    location / {
        root   /var/www/frontend;
        index  index.html;
        try_files $uri $uri/ /index.html;   # SPA 路由
    }

    # 代理后端 API（关键配置）
    location /api/ {
        proxy_pass         http://backend.com:3000/;   # 转发到后端
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        
        # 解决跨域
        add_header 'Access-Control-Allow-Origin' '*';
        add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS, PUT, DELETE';
        add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range,Authorization';
        add_header 'Access-Control-Allow-Credentials' 'true';
    }
}

```

前端请求时：

```js

// 直接请求同域地址
fetch('/api/users')        // 注意是 /api 开头
  .then(res => res.json())
  .then(data => console.log(data))

```
