# 浏览器同源策略

## 同源策略

浏览器同源策略（Same-Origin Policy） 是浏览器最核心、最基础的安全策略之一。

同源策略是指：浏览器限制一个源（Origin）中的文档或脚本与另一个源的资源进行交互。

**简单说：只有当两个 URL 的协议、域名、端口完全一致时，才算“同源”，才能无限制地互相访问对方的 DOM、Cookie、LocalStorage、发起 AJAX 请求等。**

**协议 + 域名 + 端口 三者完全相同才算同源。**

## 为什么要有同源策略

根本目的：保护用户数据安全，防止恶意网站窃取用户信息。

假设用户已经登录了银行网站 `bank.com`，浏览器保存了该网站的 Cookie（包含登录态）。

这时用户不小心打开了一个恶意网站 `evil.com`：

如果没有同源策略，`evil.com` 可以做到以下可怕的事情：

- 读取银行网站的 `Cookie` → 直接盗取登录态
- 读取银行网站的 `DOM` → 窃取银行卡号、余额等敏感信息
- 向银行网站发起 `AJAX` 请求 → 以用户身份转账、修改密码等
- 读取银行网站的 `LocalStorage` → 窃取用户数据



同源策略就是浏览器的一道安全防线，它规定：

- 不同源的网站不能互相读取对方的 Cookie、LocalStorage、IndexedDB
- 不同源的网站不能互相操作对方的 DOM
- 不同源的网站默认不能发起 AJAX 请求



## 同源策略的具体限制

1. Cookie 限制
   
   - 默认情况下，`a.com `设置的 Cookie，只有 `a.com` 及其子域名才能读取
   - `b.com` 无法通过 JS 读取 `a.com` 的 Cookie

2. DOM 限制
   
   - `a.com` 嵌入的 iframe 中的 `b.com` 页面，无法通过 JS 访问 `a.com` 主页面的 DOM

3. AJAX / Fetch 请求限制
   
   - 默认情况下，`a.com` 只能向 `a.com` 发送 AJAX 请求
   - 向其他域名发起跨域请求，默认会被浏览器拦截

加载图片、css、js可以无视同源策略

```html

<img src="跨域图片的地址" />

<link href="跨域的css地址" />

<script src="跨域的js地址"></script>

```
