# 寄生组合式继承

## 寄生组合式继承（ES5 最佳实践）

```js

function inherit(Child, Parent) {
  const F = function() {}
  F.prototype = Parent.prototype
  Child.prototype = new F()
  Child.prototype.constructor = Child
  Child.super = Parent // 可选，方便调用 super
}

function Parent() {}
function Child() {
  Parent.call(this)
}
inherit(Child, Parent)

```


现代等价写法（最推荐）


```js

function inherit(Child, Parent) {
  Child.prototype = Object.create(Parent.prototype)
  Child.prototype.constructor = Child
}

```

## 为什么要有中间函数

为什么要用一个空的中间函数 F，来写 Child.prototype = new F()？

核心原因是为了安全地、干净地实现原型继承，同时避免直接调用 new Parent() 可能带来的副作用。


![LOGO](/public/image/base/ScreenShot_2026-03-09_155553_366.png)


- 为什么直接 Child.prototype = Parent.prototype 不行？

```js
Child.prototype = Parent.prototype;
Child.prototype.constructor = Child;
```

看起来很干净，但有个致命问题：

```js
// 修改了父原型
Child.prototype.newMethod = function() { ... };

```

这行代码会直接修改 Parent.prototype！

所有 Parent 的实例都会突然多出一个 newMethod，破坏了继承的封装性。


- 为什么 new Parent() 不安全？

```js

Child.prototype = new Parent();

```

很多 Parent 构造函数是有副作用的：

```js

function Parent() {
  this.id = Math.random();
  console.log("Parent 初始化了");     // 副作用1：打印
  document.body.style.background = "red";  // 副作用2：改页面
  ajax.send(...);                       // 副作用3：发请求
}

```


只要你写 Child.prototype = new Parent()，上面这些代码就会立刻执行一次，这几乎总是我们不想要的。


- 所以中间空函数 F 的真正作用是：

    - 创建一个全新的、空的中间对象
    - 把这个对象的 __proto__ 指向 Parent.prototype
    - 这样得到的实例只有原型链，没有任何来自 Parent 构造函数的副作用
    - 再把这个干净的、有正确原型链的对象赋值给 Child.prototype


```text

写法1（有副作用）：
Child.prototype → (new Parent实例) → Parent.prototype
                     ↑ 有自己的属性、有副作用

写法2（经典寄生组合式）：
Child.prototype → (new F实例) → Parent.prototype
                     ↑ 纯净的、空的、只有原型链

```


- 等价写法

```js

// 写法一（最常用、最清晰）
function inherit(Child, Parent) {
  Child.prototype = Object.create(Parent.prototype);
  Child.prototype.constructor = Child;
}

// 写法二（等价，但更直白）
Child.prototype = Object.create(Parent.prototype, {
  constructor: {
    value: Child,
    enumerable: false,
    writable: true,
    configurable: true
  }
});

```



使用中间函数 F（或 Object.create）是为了在不执行 Parent 构造函数、不污染 Parent.prototype 的前提下，让 Child.prototype 的原型链正确指向 Parent.prototype。
