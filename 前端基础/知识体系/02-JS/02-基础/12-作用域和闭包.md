# 作用域和闭包



## 作用域

一、作用域（Scope）：变量的 “可见范围”


**作用域就是变量的“可见范围”和“生命周期”**，简单说：你在代码的哪个位置能访问到哪个变量，就是由作用域决定的。


![LOGO](/public/image/base/ScreenShot_2026-03-10_093409_297.png)


## 核心规则

作用域链（内层找外层，外层找不到内层），当代码访问一个变量时，JS 会按「当前作用域 → 外层作用域 → 全局作用域」的顺序查找，直到找到或报错（ReferenceError）。


```js

// 全局作用域
const globalVar = '全局变量';

function outer() {
  // outer 函数作用域
  const outerVar = '外层变量';

  function inner() {
    // inner 函数作用域
    const innerVar = '内层变量';
    
    // 内层能访问自身、外层、全局
    console.log(innerVar); // 内层变量
    console.log(outerVar); // 外层变量
    console.log(globalVar); // 全局变量
  }
  inner();

  // 外层不能访问内层变量（报错）
  console.log(innerVar); // ReferenceError: innerVar is not defined
}
outer();

// 全局不能访问函数内变量（报错）
console.log(outerVar); // ReferenceError: outerVar is not defined


```

- 先找自己作用域 → 有就用，没有就往外层找

- 往外找一直到全局 → 全局都没有就报错（ReferenceError）

- let/const 形成的块级作用域是“独立小监狱”，里面的变量外面看不到




## 闭包

闭包就是“函数 + 函数能记住出生时周围的环境”。


当一个函数能够“记住并访问”它被创建时所处的词法作用域，即使这个函数在它定义的作用域之外被调用，它也能访问那个作用域里的变量，这个现象就叫闭包。


- 闭包的形成条件（3 个）

    - 存在嵌套函数（内层函数 + 外层函数）；
    - 内层函数引用了外层函数的变量 / 参数；
    - 内层函数被外层作用域之外的地方调用（如返回、赋值给全局变量）。



```js

function createCounter() {
  let count = 0; // count 被“记住”了
  // 内层函数被返回（保存到外层作用域之外）
  return function() {
    // 内层函数：引用外层变量 count
    count++;
    console.log(count);
  };
}

const counter1 = createCounter();
const counter2 = createCounter();

counter1(); // 1
counter1(); // 2
counter2(); // 1   ← 独立的计数器
```

- 为什么 count 没有被销毁？

    - createCounter 执行完后，它的执行上下文本该销毁
    - 但内部返回的匿名函数还在被外部变量 counter1 引用
    - 这个匿名函数记住了自己出生时的环境（包括 count 变量）
    - 只要函数还活着，它记住的环境就不能被销毁 → 形成了闭包



## 闭包的三大组成

- 外部函数：创建了私有变量和返回内部函数 `createCounter`

- 私有变量：被外部函数声明，但内部函数能访问 `let count = 0`

- 内部函数：能访问外部函数的变量，并在外部被使用 `return function() { count++ }`



## 函数作为返回值 vs 函数作为参数

 自由变量的查找，是在函数定义的地方，向上级作用域查找，不是在执行的地方

## 闭包的三个核心用途

- （1）保存私有变量（最常用）

实现 “私有属性”—— 变量只能通过特定方法访问 / 修改，避免全局污染。

```js

// 计数器模块（闭包实现私有变量）
function createCounter() {
  let count = 0; // 私有变量，外部无法直接访问
  return {
    increment: () => count++,
    decrement: () => count--,
    getCount: () => count
  };
}

const counter1 = createCounter();
counter1.increment();
console.log(counter1.getCount()); // 1
counter1.count = 100; // 直接赋值无效（count 是私有变量）
console.log(counter1.getCount()); // 1

const counter2 = createCounter(); // 每个实例的私有变量独立
console.log(counter2.getCount()); // 0

```



- （2）防抖 / 节流（前端高频场景）

利用闭包保存定时器变量，实现防抖（避免频繁触发）：


```js

function debounce(fn, delay) {
  let timer = null; // 闭包保存定时器，多次调用共享同一个 timer
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 搜索框防抖
const search = debounce((keyword) => {
  console.log(`搜索：${keyword}`);
}, 500);

// 连续输入时，只有停止 500ms 后才执行
search('j');
search('ja');
search('jav');
search('javascript'); // 最终只执行这一次


```


- （3）柯里化（参数复用）

结合闭包实现参数分步传递


```js

function curryAdd(a) {
  // 闭包保存第一个参数 a
  return function(b) {
    return a + b;
  };
}

const add5 = curryAdd(5); // 保存 a=5
console.log(add5(3)); // 8（复用 a=5，传 b=3）
console.log(add5(10)); // 15（复用 a=5，传 b=10）

```


## 作用域 vs 闭包


![LOGO](/public/image/base/ScreenShot_2026-03-10_142856_358.png)






- 作用域：变量的可访问范围，核心是「作用域链（内层找外层）」，分全局 / 函数 / 块级作用域；
- 闭包：内层函数引用外层变量且被外层之外调用，导致外层变量不销毁；
- 闭包核心价值：保存私有变量、防抖节流、参数复用（柯里化）；
- 关键坑点：闭包可能导致内存泄漏，需手动解除引用。
- 作用域规定了变量的访问规则，闭包是利用这个规则，让内层函数 “锁住” 外层变量不被销毁。
- 作用域决定变量能被谁看到，闭包让函数即使跑到别的地方，也能“带着出生时的环境”继续使用那些变量。




## 实际开发中闭包的应用场景，举例说明

### 1. 封装私有变量（最常用！防全局污染）

不让外部直接修改变量，只能通过暴露的方法操作。


```js

function createCache() {
  // 闭包私有变量，外部绝对访问不到
  const data = {};

  return {
    get(key) {
      return data[key];
    },
    set(key, value) {
      data[key] = value;
    }
  };
}

const cache = createCache();
cache.set('name', '闭包');
console.log(cache.get('name')); // 闭包
console.log(cache.data); // undefined（安全！）

```

### 2. 防抖（debounce）高频必用

搜索框、窗口 resize、按钮点击防止重复触发。

```js

function debounce(fn, delay = 300) {
  // 闭包保存 timer，不被销毁
  let timer = null;

  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 使用
const search = debounce((keyword) => {
  console.log('请求接口：', keyword);
}, 500);

search('vue');
search('vuex'); // 只执行最后一次

```


### 3. 节流（throttle）高频必用

滚动、拖拽、高频点击限制频率。

```js

function throttle(fn, interval = 500) {
  // 闭包保存上次时间
  let lastTime = 0;

  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

```

### 4. 函数柯里化（参数复用）

提前传一部分参数，后面复用。工具函数、表单校验、类型判断。

```js

function checkType(type) {
  // 闭包保存 type
  return (val) => Object.prototype.toString.call(val) === `[object ${type}]`;
}

const isString = checkType('String');
const isArray = checkType('Array');

console.log(isString('abc')); // true
console.log(isArray([])); // true


```

### 5. 循环中保存变量（经典面试 + 实际用）

解决 var 异步取值错误。现在用 let 替代，但原理依然是闭包！

```js

for (var i = 0; i < 5; i++) {
  // 闭包保存每次的 i
  (function (i) {
    setTimeout(() => console.log(i), 1000);
  })(i);
}
// 输出 0 1 2 3 4

```

### 6. Vue / React 组件内保存状态

:::code-group


```vue

<script>
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    add() {
      // 闭包访问组件实例的 count
      this.count++;
    }
  }
}
</script>

```



```jsx

function Counter() {
  const [count, setCount] = useState(0);

  const add = () => {
    // 闭包捕获 count 状态
    setCount(count + 1);
  };

  return <button onClick={add}>{count}</button>;
}

```

:::



闭包的核心作用总结（一句话）
- 闭包 = 函数 + 它能访问的外部作用域它能让本该销毁的变量继续存活，从而实现：
  - 私有变量
  - 缓存
  - 防抖节流
  - 柯里化
  - 模块化
  - 组件状态保存
