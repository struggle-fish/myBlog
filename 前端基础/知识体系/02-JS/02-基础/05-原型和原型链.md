# 原型和原型链



## class 和继承

### class 是什么？怎么用？

class 是创建对象的 “模板”，它封装了对象的属性和方法。

```js

// 定义一个 Person 类
class Person {
  // 1. constructor：构造方法（必须），new 时自动调用
  constructor(name, age) {
    this.name = name; // this 指向实例对象
    this.age = age;
  }

  // 2. 实例方法：实例可以直接调用
  sayHi() {
    console.log(`大家好，我是${this.name}，今年${this.age}岁`);
  }

  // 3. 静态方法：用 static 修饰，只能通过类调用，实例不能调用
  static species() {
    console.log('我是人类');
  }

  // 4. getter/setter：控制属性的读取和赋值
  get age() {
    return this._age; // 用 _age 存储实际值（避免和 getter/setter 重名）
  }

  set age(value) {
    if (value < 0 || value > 150) {
      throw new Error('年龄不合法');
    }
    this._age = value;
  }
}

// 使用：new 一个实例
const p1 = new Person('张三', 18);
p1.sayHi(); // 大家好，我是张三，今年18岁（调用实例方法）
Person.species(); // 我是人类（调用静态方法）
console.log(p1.age); // 18（调用 getter）
p1.age = 20; // 调用 setter

```


### class的核心注意点

- class 必须用 new 调用：不能直接 Person()，会报错；

- class 内部默认是严格模式；

- 实例方法在原型上：p1.sayHi === Person.prototype.sayHi（节省内存）；

- 静态方法在类上：只能通过 Person.species() 调用，p1.species() 会报错；

- getter/setter 不是方法，是属性访问器：调用时不加括号（p1.age 不是 p1.age()）。



### class 本质

ES6 的 class 只是让写法更简洁，底层依然是 ES5 的构造函数 + 原型链

::: code-group

```js [ES5 写法]

// 父类
function Person(name, age) {
  this.name = name;
  this.age = age;
}
Person.prototype.sayHi = function() {
  console.log(`大家好，我是${this.name}`);
};

// 子类
function Student(name, age, grade) {
  Person.call(this, name, age); // 借用父类构造函数
  this.grade = grade;
}
// 原型链继承
Student.prototype = Object.create(Person.prototype);
Student.prototype.constructor = Student;
// 重写方法
Student.prototype.sayHi = function() {
  Person.prototype.sayHi.call(this);
  console.log(`我是${this.grade}年级的学生`);
};


```


```js [ES6 写法]
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  sayHi() {
    console.log(`大家好，我是${this.name}`);
  }
}

class Student extends Person {
  constructor(name, age, grade) {
    super(name, age);
    this.grade = grade;
  }
  sayHi() {
    super.sayHi();
    console.log(`我是${this.grade}年级的学生`);
  }
}

// class 只是把 ES5 的复杂写法封装成了更简洁的语法，底层逻辑完全一样。
```

### class 继承基础

继承是让一个类（子类）拥有另一个类（父类）的属性和方法，并可以扩展自己的功能。ES6 用 `extends` 关键字实现继承，用 `super `函数调用父类


```js

// 父类：Person
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  sayHi() {
    console.log(`大家好，我是${this.name}`);
  }
}

// 子类：Student，继承 Person
class Student extends Person {
  constructor(name, age, grade) {
    // 1. super()：调用父类的 constructor（必须在 constructor 第一行调用！）
    // 必须先调用 super()，否则 this 会报错
    super(name, age); // 把 name 和 age 传给父类构造函数
    this.grade = grade; // 子类自己的属性
  }

  // 2. 重写父类方法：子类可以定义和父类同名的方法，覆盖父类
  sayHi() {
    // 2.1 super.方法名()：调用父类的方法
    super.sayHi(); // 先调用父类的 sayHi
    console.log(`我是${this.grade}年级的学生`); // 再扩展自己的逻辑
  }

  // 3. 子类自己的方法
  study() {
    console.log(`${this.name}在学习`);
  }
}

// 使用子类
const s1 = new Student('李四', 16, 3);
s1.sayHi(); // 大家好，我是李四；我是3年级的学生（重写后的方法）
s1.study(); // 李四在学习（子类自己的方法）
console.log(s1.name); // 李四（继承自父类的属性）


```

**super的两种用法**

- `super(...)` 作为函数调用，调用父类的 constructor

- `super.方法名()` 作为对象使用，调用父类的方法


### 继承的核心注意点

- 子类必须在 constructor 中调用 `super()`：因为子类没有自己的 this 对象，需要继承父类的 this；如果不调用 `super()`，this 是 undefined，会报错；

- 子类可以重写父类方法：定义同名方法即可覆盖；

- 子类继承父类的所有属性和方法：包括实例方法、静态方法（静态方法也会继承！）；

- instanceof 可以判断继承关系：`s1 instanceof Student → true，s1 instanceof Person → true`。

- extends 实际上是把子类的 `__proto__ `指向父类（构造函数继承）

- 子类的 `prototype.__proto__ `指向父类的 prototype（方法继承）

- 必须在子类 constructor 里先写 `super()` 才能使用 this

- super() 相当于调用父类的 constructor

- `super.xxx()` 调用父类的同名方法


```text
Student (构造函数)
   ↓
Student.prototype ── __proto__ ──> Person.prototype
   ↑                                       ↑
   └─────────── __proto__ ─────────────────┘

```



## 继承的方式

### 1. 原型链继承（最基础）

核心原理：让子类的 `prototype` 指向父类的`实例`，通过原型链访问父类属性和方法

`Child.prototype` 指向 `Parent` 的实例，`Child` 的实例通过原型链访问 `Parent.prototype` 的属性和方法。

```js

// 父类
function Parent() {
  this.name = 'Parent';
  this.hobbies = ['读书', '运动']; // 引用类型属性
}
Parent.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

// 子类
function Child() {}
// 📢 核心：子类原型指向父类实例
Child.prototype = new Parent();
// 修正 constructor 指向（否则会指向 Parent）
Child.prototype.constructor = Child;

// 测试
const c1 = new Child();
const c2 = new Child();

```

优点：实现简单，易于理解，继承了父类的原型方法和属性


缺点（致命）：

1. 父类的引用属性会被所有子类实例共享：


```js
c1.hobbies.push('编程');
// ['读书', '运动', '编程']（c2 也被影响了！）
console.log(c2.hobbies); 

```

2. 创建子类实例时，无法向父类构造函数传参

```js
// 无法给 Parent 传 name 参数
const c3 = new Child('张三'); 
```

3. 共享原型问题：所有 `Child` 实例共享同一个原型对象，修改 `Child.prototype` 的引用类型属性会影响所有实例。

使用场景：仅用于理解原型链，实际开发不推荐


### 2.构造函数继承

核心原理：在子类构造函数中，通过 call()/apply() 调用父类构造函数，绑定 this 到子类实例。


```js

// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['读书', '运动'];
}
Parent.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

// 子类
function Child(name, age) {
  // 核心：借用父类构造函数，绑定 this 到子类实例
  Parent.call(this, name); // 可以传参！
  this.age = age; // 子类自己的属性
}

// 测试
const c1 = new Child('张三', 18);
const c2 = new Child('李四', 20);

```

优点： 

- 避免引用属性共享，每个 Child 实例有独立的属性副本

```js
c1.hobbies.push('编程');
console.log(c2.hobbies); // ['读书', '运动']（c2 不受影响）
```

- 可以向父类构造函数传参

缺点（致命）：

无法继承父类原型上的方法：只能继承 Parent 构造函数中定义的属性和方法，`Parent.prototype` 上的方法不可用。

```js

c1.sayHi(); 
// 报错：c1.sayHi is not a function（原型链断了！）
```

父类方法无法复用：每个子类实例都有父类方法的副本，浪费内存。

适用场景：仅用于继承父类属性，实际开发不推荐。


### 3.组合继承（原型链+构造函数）

核心原理：结合前两种方式 —— 用构造函数继承属性，用原型链继承方法。**最常用**

```js

// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['读书', '运动'];
}
Parent.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

// 子类
function Child(name, age) {
  // 1. 构造函数继承：继承属性（第一次调用 Parent）
  Parent.call(this, name);
  this.age = age;
}
// 2. 原型链继承：继承方法（第二次调用 Parent）
// 子类原型上有name属性 而且是undefined
Child.prototype = new Parent();
Child.prototype.constructor = Child;

// 测试
const c1 = new Child('张三', 18);
const c2 = new Child('李四', 20);

```

优点：
- 结合了前两种方式的优点：引用属性不共享、可以传参、继承父类原型方法。
- 是 ES5 中最常用的继承方式。


缺点：

- 父类构造函数会被调用两次：
  - 第一次：在 new Parent() 设置原型时，new Parent() 可能引发不必要的副作用
  - 第二次：在 Parent.call 初始化实例时


- 子类原型上有多余的父类属性：原型对象包含多余的父类实例属性（由 new Parent() 产生），尽管通常被实例属性覆盖

```js
// undefined（多余的属性，浪费内存）
console.log(Child.prototype.name); 
```

适用场景：

ES5 环境下的通用继承，实际开发推荐（但有更优的寄生组合式继承）。





### 4.原型式继承

核心原理：基于现有对象创建新对象，不需要构造函数（本质是浅拷贝）。ES5 用 `Object.create()` 实现。

```js

// 父对象（不是构造函数）
const parent = {
  name: 'Parent',
  hobbies: ['读书', '运动'],
  sayHi: function() {
    console.log(`我是${this.name}`);
  }
};

function createObject(parent) {
  function F() {}
  F.prototype = parent;
  return new F();
}
// 类似 Object.create
const child1 = createObject(parent);
const child2 = createObject(parent);

```


优点：实现简单，不需要构造函数，适合 “基于现有对象扩展” 的场景。


缺点：

- 引用属性会被共享（和原型链继承一样）：

```js
child1.hobbies.push('编程');
// ['读书', '运动', '编程']（被影响）
console.log(child2.hobbies); 
```

- 无法向父对象传参。

适用场景：

仅用于 “基于现有对象快速创建新对象”，实际开发不推荐。


### 5.寄生式继承

核心原理：在原型式继承的基础上，给新对象 “寄生”（添加）一些方法。

```js

// 父对象
const parent = {
  name: 'Parent',
  hobbies: ['读书', '运动']
};

// 核心：寄生函数，在原型式继承基础上扩展方法
function createChild(parentObj) {
  const child = Object.create(parentObj); // 原型式继承
  // 寄生：添加自己的方法
  child.sayHi = function() {
    console.log(`我是${this.name}`);
  };
  return child;
}

// 测试
const child1 = createChild(parent);
const child2 = createChild(parent);

```

优点：可以在继承的同时扩展方法

缺点：

- 引用属性共享（和原型式继承一样）；
- 方法无法复用：每个实例都有 sayHi 的副本，浪费内存。

适用场景：仅用于理解寄生思想，实际开发不推荐。




### 6.寄生组合继承

核心原理：使用 Object.create 创建一个干净的中间对象，指向 Parent.prototype，避免直接调用 new Parent()。

```js
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['读书', '运动'];
}
Parent.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

// 子类
function Child(name, age) {
  // 1. 构造函数继承：继承属性（仅调用一次 Parent）
  Parent.call(this, name);
  this.age = age;
}

// 2. 核心：寄生组合式继承原型
// 用 Object.create() 创建 Parent.prototype 的副本，赋值给 Child.prototype
Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child;

// 测试
const c1 = new Child('张三', 18);
const c2 = new Child('李四', 20);

```

优点：
- 只调用一次父类构造函数：避免了组合继承的问题；
- 原型链保持正确：没有多余的父类属性；
- 引用属性不共享、可以传参、继承父类原型方法。


缺点：写法稍微复杂一点（但可以封装成工具函数）。

适用场景：ES5 环境下的最佳继承方式，实际开发强烈推荐。

### 7. ES6 Calss + extends

核心原理：使用 extends 和 super 实现继承，底层基于寄生组合继承。

底层仍然是寄生组合式继承 + 对 constructor 的特殊处理（super 实际上是借用父类构造函数并绑定 this），浏览器或 babel 转译后可以看到类似 `Child.__proto__ = Parent` 和 `Child.prototype.__proto__ = Parent.prototype` 的实现。
```js
// 父类
class Parent {
  constructor(name) {
    this.name = name;
    this.hobbies = ['读书', '运动'];
  }
  sayHi() {
    console.log(`我是${this.name}`);
  }
  static species() { // 静态方法也会继承！
    console.log('我是人类');
  }
}

// 子类：用 extends 继承
class Child extends Parent {
  constructor(name, age) {
    super(name); // 核心：调用父类 constructor（必须在第一行）
    this.age = age;
  }
  // 重写父类方法
  sayHi() {
    super.sayHi(); // 调用父类方法
    console.log(`我今年${this.age}岁`);
  }
}

// 测试
const c1 = new Child('张三', 18);
const c2 = new Child('李四', 20);
Child.species(); // 我是人类（静态方法也继承了）

```


优点：
- 语法简洁清晰：符合传统面向对象语言的写法；
- 支持静态方法继承：ES5 继承方式很难实现静态方法继承；
- 底层是寄生组合式继承：性能最优；
- super 关键字：方便调用父类构造函数和方法。


缺点
- 需要 ES6 环境（但现代浏览器和 Node.js 都支持，可通过 Babel 转译）。

适用场景：现代开发的首选，强烈推荐。


### 8.Mixin混合

核心原理：通过复制属性或方法到目标对象的原型或实例上，实现“继承”。
```js

const Mixin = {
  say() { console.log('Mixin says'); }
};

function Child() {}
Object.assign(Child.prototype, Mixin);

```


优点：

- 灵活：可以选择性地混合多个对象的属性/方法。
- 不依赖原型链，适合横向扩展功能。

缺点：

- 命名冲突：多个 Mixin 可能导致属性覆盖。
- 维护性差：逻辑分散，难以追踪来源。
- 不支持复杂的继承关系。

使用场景：适合为对象动态添加功能的场景。

## new 的原理

new 是 JavaScript 中用于创建构造函数实例的操作符，它的核心作用是把一个普通函数变成 “构造函数”，并生成一个绑定了该函数原型的新对象
当你执行 `new Constructor(...)` 时，JavaScript 会自动完成以下 4 步：

### 创建空对象

创建一个空的新对象

```js

const obj = {};

```

### 链接原型 

链接新对象的原型到构造函数的 prototype

让新对象的 `__proto__`（隐式原型）指向构造函数的 prototype（显式原型），这样新对象就能访问构造函数原型上的方法了。

```js
obj.__proto__ = Constructor.prototype;

```

### 绑定 this 并执行构造函数 

绑定 this 到新对象，执行构造函数

把构造函数的 this 指向新创建的对象，然后执行构造函数的代码（给新对象添加属性）。

```js

const result = Constructor.apply(obj, arguments);

```

### 判断返回值 

判断构造函数的返回值，决定返回什么

如果构造函数返回一个对象（{}、[]、function 等），则返回这个对象；

如果构造函数返回基本类型（number、string、boolean、null、undefined），则忽略返回值，返回新创建的对象。


```js

return typeof result === 'object' && result !== null ? result : obj;

```

用一个具体的 Person 构造函数，一步步看 new 的执行过程：

```js

// 定义构造函数
function Person(name, age) {
  this.name = name; // this 会被绑定到新对象
  this.age = age;
  // 构造函数可以有返回值（可选）
  // return { hobby: 'coding' }; // 如果返回对象，new 会返回这个对象
  // return 100; // 如果返回基本类型，new 会忽略，返回新对象
}

// 给构造函数的原型添加方法
Person.prototype.sayHi = function() {
  console.log(`大家好，我是${this.name}`);
};

// 使用 new 创建实例
const p1 = new Person('张三', 18);

// 创建空对象：const obj = {}；
// 链接原型：obj.__proto__ = Person.prototype（所以 p1.sayHi 能访问到原型上的方法）；

// 绑定 this 并执行构造函数：
// Person.apply(obj, ['张三', 18]) → obj.name = '张三'，obj.age = 18；


// 判断返回值：Person 没有显式返回对象，所以返回 obj → p1 = obj。


```

### 实现一个new

```js

function myNew(Constructor, ...args) {
  // 1. 创建空对象
  const obj = {};

  // 2. 链接原型（等价于 obj.__proto__ = Constructor.prototype）
  Object.setPrototypeOf(obj, Constructor.prototype);

  // 直接创建一个原型已链接好的对象（合并 1、2 步）
  // const obj = Object.create(Constructor.prototype);

  // 3. 绑定 this 并执行构造函数
  const result = Constructor.apply(obj, args);

  // 4. 判断返回值
  return typeof result === 'object' && result !== null ? result : obj;
}


// 用刚才的 Person 构造函数测试  
const p2 = myNew(Person, '李四', 20);
console.log(p2.name); // 李四
console.log(p2.age); // 20
p2.sayHi(); // 大家好，我是李四
console.log(p2.__proto__ === Person.prototype); // true
```

::: tip

1. 构造函数返回值的影响

如果构造函数显式返回一个对象，new 会返回这个对象，而不是新创建的对象：

```js
function Person(name) {
  this.name = name;
  return { hobby: 'coding' }; // 显式返回对象
}

const p = new Person('张三');
// undefined（返回的是 { hobby: 'coding' }，不是新对象）
console.log(p.name); 
console.log(p.hobby); // coding

```


2. 箭头函数不能作为构造函数

因为箭头函数没有自己的 this，也没有 prototype，所以不能用 new 调用：

```js

const Person = (name) => { this.name = name; };
const p = new Person('张三'); 
// 报错：TypeError: Person is not a constructor
```

:::



## 原型和原型链

![LOGO](/public/image/base/36409-369763c6cb6d47bcae20146cca445470.png)

JavaScript 里的一切对象（包括函数本身）最终都通过 `__proto__ `连到 Object.prototype，而函数额外有一条 prototype 链路，形成一个闭环结构。

### 三类角色

1. 构造函数（Functions）
    - 图中 function Foo()、function Object()、function Function() 都是构造函数。
    - 它们是 “用来造对象的工厂”，本身也是对象。
2. 原型对象（Prototypes）
    - 每个构造函数都有一个 prototype 属性，指向它的 “原型对象”，如 Foo.prototype、Object.prototype、Function.prototype。
    - 原型对象是 “共享方法和属性的仓库”，所有由该构造函数创建的实例，都会共享这个仓库里的东西。
3. 实例对象（Instances）
    - 图中 f1、f2（new Foo()）、o1、o2（new Object()）都是实例对象。
    - 每个实例对象都有一个内部的 `[[Prototype]]`（在浏览器中常写作 `__proto__`），指向创建它的构造函数的 prototype。


### 核心关系链（看图里的箭头）

1. 实例 → 原型 → 根原型

    - `f1.__proto__ → Foo.prototype`
    - `Foo.prototype.__proto__ → Object.prototype`
    - `Object.prototype.__proto__ → null`（原型链的终点）

这就是我们常说的原型链：当你访问 `f1.toString()` 时，JS 会先在 f1 自身找，找不到就去 Foo.prototype 找，再找不到就去 Object.prototype 找，直到 null。


2. 构造函数 ↔ 原型对象

    - Foo.prototype.constructor → Foo
    - Object.prototype.constructor → Object
    - Function.prototype.constructor → Function

这是一个双向绑定：
- 构造函数通过 prototype 找到它的原型对象。
- 原型对象通过 constructor 指回它的构造函数，用来标识 “我是谁的原型”。


3. 所有函数都是 Function 的实例
    - `Foo.__proto__ → Function.prototype`
    - `Object.__proto__ → Function.prototype`
    - `Function.__proto__ → Function.prototype`（自己是自己的实例，这是 JS 里一个很特殊的点）


所有函数（包括 Object、Function 本身）都继承了 Function.prototype 上的方法，比如 call()、apply()、bind()。

4. 所有原型对象最终都是 Object 的实例

    - `Foo.prototype.__proto__ → Object.prototype`
    - `Function.prototype.__proto__ → Object.prototype`

这解释了为什么，任何对象（包括函数、原型对象）都能调用 toString()、hasOwnProperty() 等方法，因为这些方法都定义在 Object.prototype 上。

### 小结

- 万物皆对象，所有对象最终都继承自 Object.prototype。
- 函数作为普通对象：通过 `__proto__ `连到 Function.prototype（因为函数是由 Function 创建的）
- 函数作为构造函数：通过 prototype 属性提供给 new 出来的实例用
- 实例通过 `__proto__ `找爸爸（原型），构造函数通过 prototype 找孩子（共享属性），形成了一条从实例到根原型的链条。


```text
普通实例 → __proto__ → 构造函数的 prototype → __proto__ → Object.prototype → null

而所有构造函数（包括 Object 和 Function）本身的 __proto__ → Function.prototype → Object.prototype → null

```

::: tip

“万物皆对象，函数生万物；
prototype 给子用，proto 找爹去；
Function 自己生自己，Object 是所有爹的爹。”

:::


```js

function Foo() {}
const f1 = new Foo();

console.log(f1.__proto__ === Foo.prototype); // true
console.log(Foo.prototype.__proto__ === Object.prototype); // true
console.log(Foo.__proto__ === Function.prototype); // true
console.log(Function.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null

```




## 常见面试题

### 如何准确判断一个变量是不是数组

#### 1. Array.isArray

核心原理：ES5 新增的数组静态方法，专门用于判断值是否为数组，无视原型链、跨窗口 /iframe 都能准确判断。

```js
// 正确判断
console.log(Array.isArray([])); // true
console.log(Array.isArray([1,2,3])); // true
console.log(Array.isArray(new Array())); // true

// 边界判断（全部 false）
console.log(Array.isArray({})); // false
console.log(Array.isArray('123')); // false
console.log(Array.isArray(null)); // false
console.log(Array.isArray(undefined)); // false
console.log(Array.isArray(123)); // false
Array.isArray(arguments)      // false（类数组）
Array.isArray(document.querySelectorAll('div'))  // false（NodeList）

```

优点：

- ES5 标准方法，所有现代浏览器（包括 IE9+， Node.js）都支持
- 能正确区分 真正的数组 和 类数组（arguments、NodeList 等）
- 解决了 instanceof 跨窗口 /iframe 失效的问题
- 写法最简洁、最清晰

缺点：

- 不支持 ES5 之前的环境（如 IE8 及以下，可 polyfill 兼容）。

```js

if (!Array.isArray) {
  Array.isArray = function(obj) {
    return Object.prototype.toString.call(obj) === '[object Array]';
  };
}

```

#### 2. Object.prototype.toString.call

核心原理：利用 Object.prototype.toString 会返回值的内部 [[Class]] 标签（格式：[object 类型名]），数组的标签是 [object Array]，不受原型链、跨窗口影响。


```js

Object.prototype.toString.call(变量) === '[object Array]'

// 封装成工具函数（推荐）
function isArray(obj) {
  return Object.prototype.toString.call(obj) === '[object Array]';
}

// 测试
console.log(isArray([])); // true
console.log(isArray(new Array())); // true
console.log(isArray({})); // false
console.log(isArray(null)); // false（返回 [object Null]）
console.log(isArray(undefined)); // false（返回 [object Undefined]）

```

优点：

- 非常可靠，几乎不可能被伪造
- 能区分 Array、Arguments、NodeList、HTMLCollection 等
- IE6/7/8 完全兼容

缺点：写法比 Array.isArray() 繁琐，语义性稍弱。

#### 3. instanceof

instanceof Array（不推荐跨窗口场景）

核心原理： 判断 Array.prototype 是否存在于变量的原型链上（即变量是否是 Array 构造函数的实例）。


```js

console.log([] instanceof Array); // true
console.log(new Array() instanceof Array); // true
console.log({} instanceof Array); // false

```

优点：写法简洁，ES3 就支持，兼容旧环境。

致命缺点：

- 跨窗口 /iframe 失效：不同窗口的 Array 构造函数是不同的实例，原型链不互通；

```js

// 假设有两个 iframe，iframe1 的数组传到 iframe2
const arr = iframe1.window.Array(1,2,3);
console.log(arr instanceof iframe2.window.Array); // false（错误！实际是数组）

```

- 原型链被修改时失效：

```js

const arr = [];
arr.__proto__ = Object.prototype; // 修改原型链
console.log(arr instanceof Array); // false（错误！实际是数组）

```



#### 4. constructor

核心原理：数组实例的 constructor 指向 Array 构造函数。

```js

console.log([].constructor === Array); // true

```

缺点：

- constructor 可被手动篡改，导致判断错误：

```js

const arr = [];
arr.constructor = Object; // 篡改 constructor
console.log(arr.constructor === Array); // false（错误）

```

### class 的原型本质，怎么理解

class 是 ES5 「构造函数 + 原型链」的语法糖，它的所有特性（实例方法、静态方法、继承）最终都会映射到原型链上，底层逻辑完全一致。

#### 4类成员

一个完整的 class 包含 4 类成员，每一类都有明确的原型链映射：


![LOGO](/public/image/base/ScreenShot_2026-03-09_204830_304.png)


```js
// ES6 class 写法
class Person {
  // 1. constructor：构造函数（对应 ES5 构造函数本身）
  constructor(name) {
    this.name = name; // 实例属性（挂载到 this 上，不在原型链）
  }

  // 2. 实例方法：挂载到 Person.prototype 上
  sayHi() {
    console.log(`我是${this.name}`);
  }

  // 3. 静态方法：挂载到 Person 类本身（构造函数上）
  static species() {
    console.log('人类');
  }
}

// 创建实例
const p = new Person('张三');

```


对应原型链关系（关键！）

```text

// 1. 实例 p 的原型链 new Person() 创建的实例，原型指向 Person.prototype
p.__proto__ === Person.prototype → true
p.name → 张三（实例自身属性，不在原型链）
p.sayHi === Person.prototype.sayHi → true（实例方法在原型上）

// 2. 静态方法的位置
Person.species === Person.prototype.species → false（静态方法不在原型上）
Person.species → function() { ... }（直接挂载到类上）

// 3. 原型链末端 Person.prototype 是一个普通对象
Person.prototype.__proto__ === Object.prototype → true

// Person 本身是一个函数，而所有函数都是 Function 的实例
Person.__proto__ === Function.prototype

// 原型对象上的 constructor 属性指回构造函数
Person.prototype.constructor === Person

```


结论：class 没有任何魔法，它只是把原型链操作包装成了更像传统面向对象语言的写法。


#### class vs ES5 原型写法（逐行对比）

把上面的 class 转成 ES5 原型写法，你会发现完全一一对应：

```js

// ES5 等价写法（class 的底层逻辑）
function Person(name) {
  // 对应 class 的 constructor
  this.name = name;
}

// 对应 class 的实例方法 sayHi
Person.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

// 对应 class 的静态方法 species
Person.species = function() {
  console.log('人类');
};

// 创建实例（和 class 完全一样）
const p = new Person('张三');


// class 写法的实例
const p1 = new Person('张三');
// ES5 写法的实例
const p2 = new Person('李四');

console.log(p1.__proto__ === p2.__proto__); // true（共享同一个原型）
console.log(p1.sayHi === Person.prototype.sayHi); // true
console.log(Person.species === Person.prototype.species); // false

```


#### class 继承的原型本质（extends + super）


```js

class Student extends Person {
  constructor(name, grade) {
    super(name)
    this.grade = grade
  }
}

```

引擎实际做了三件事：

- 静态成员继承 `Student.__proto__ = Person`

- 原型链继承Student.prototype = Object.create(Person.prototype)
    - → 让 Student.prototype 的原型指向 Person.prototype
    - → 同时修复 Student.prototype.constructor = Student
- super() 的本质super(name) → 其实就是 Person.call(this, name)
    - 把父类的实例属性绑定到子类的 this 上


#### 为什么说 class 是“语法糖”？

class Child extends Parent 的底层是「寄生组合式继承」，核心是修改原型链的指向：

::: code-group

```js [class]
class Parent {
  constructor(name) {
    this.name = name;
  }
  sayHi() { console.log(`我是${this.name}`); }
}

class Child extends Parent {
  constructor(name, age) {
    super(name); // 调用父类 constructor
    this.age = age;
  }
  study() { console.log('学习'); }
}

const c = new Child('李四', 18);

```

```js [寄生组合]

function Parent(name) {
  this.name = name;
}
Parent.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

function Child(name, age) {
  Parent.call(this, name); // 对应 super(name)
  this.age = age;
}

// 核心：子类原型 __proto__ 指向父类原型（对应 extends）
Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child; // 修正 constructor 指向

// 子类实例方法
Child.prototype.study = function() {
  console.log('学习');
};

const c = new Child('李四', 18);

```

```js [原型链]

c（实例）
├── __proto__ → Child.prototype
│   ├── study: function() {}（子类实例方法）
│   ├── constructor: Child
│   └── __proto__ → Parent.prototype
│       ├── sayHi: function() {}（父类实例方法）
│       ├── constructor: Parent
│       └── __proto__ → Object.prototype
├── name: '李四'（实例自身属性）
└── age: 18（实例自身属性）

```

:::


#### 快速记忆


- class = 构造函数 + 原型方法 的语法糖

- constructor = 函数体

- 普通方法 → 自动放 prototype 上

- static 方法 → 放构造函数本身上

- extends = 寄生组合式继承 的官方写法

- super() = Parent.call(this, ...)

- 所有 class 最终都逃不过 `__proto__` 和 prototype 这两条链



```js
class A {}
class B extends A {}

console.log(B.__proto__ === A)                  // true
console.log(B.prototype.__proto__ === A.prototype) // true
console.log(new B().__proto__ === B.prototype)  // true
console.log(A.prototype.constructor === A)     // true
```
