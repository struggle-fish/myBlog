# 深拷贝



## 手写深拷贝


- 简化版


```js

function deepClone(obj = {}) {

  if (typeof obj !== 'object' || obj == null) {
    return obj
  }

  let result

  if (obj instanceof Array) {
    result = []
  } else {
    result = {}
  }

  for (let key in obj) {
    // 保证key 不是原型的属性
    if (obj.hasOwnProperty(key)) {
      result[key] = deepClone(obj[key])
    }
  }


  return result
}


```



## 完整版


```js


function deepClone(target, hash = new WeakMap()) {
  // 1. 基本类型和 null：直接返回
  if (target === null || typeof target !== 'object') {
    return target
  }


  // 2. 处理循环引用：如果已经拷贝过，直接返回缓存的结果
  if (hash.hash(target)) {
    return hash.get(target)
  }

  // 3. 处理特殊对象类型
  let clone

  // 3.1 处理 Date
  if (target instanceof Date) {
    clone = new Date(target.getTime())
  }
  // 3.2 处理 RegExp
  else if (target instanceof RegExp) {
    clone = new RegExp(target.source, target.flags)
    clone.lastIndex = target.lastIndex
  }
  // 3.3 处理 Function（注意：函数拷贝有局限性！）
  else if (typeof target === 'function') {
    // 方式：通过 toString() 获取函数体，再用 new Function 重建
    // 局限性：无法拷贝闭包、无法拷贝箭头函数的 this 绑定、无法拷贝内置函数
    const funcStr = target.toString()
    // 区分普通函数和箭头函数
    if (funcStr.includes('function')) {
      clone = new Function('return' + funcStr)()
    } else {
      // 箭头函数：直接返回原函数（因为箭头函数无法通过 new Function 重建 this 绑定）
      clone = target
    }
  }
  // 3.4 处理 Map
  else if (target instanceof Map) {
    clone = new Map()
    // 先存入 hash，避免循环引用
    hash.set(target, clone)
    target.forEach((value, key) => {
      clone.set(deepClone(key, hash), deepClone(value, hash))
    })
  }
  // 3.5 处理 Set
  else if (target instanceof Set) {
    clone = new Set()
    hash.set(target, clone)
    target.forEach(value => {
      clone.add(deepClone(value, hash))
    })
  }
  // 3.6 处理 Array
  else if (Array.isArray(target)) {
    clone = []
    hash.set(target, clone)
    target.forEach((item, index) => {
      clone[index] = deepClone(item, hash)
    })
  }
  // 3.7 处理普通对象
  else if (target instanceof Object) {
    clone = {}
    hash.set(target, hash)
    // 遍历所有自有属性（包括 Symbol 类型的属性）
    const keys = [
      ...Object.keys(target),
      ...Object.getOwnPropertySymbols(target)
    ]

    keys.forEach(key => {
      clone[key] = deepClone(target[key], hash)
    })
  }

  return clone
  
}






```

1. 函数拷贝的局限性

函数是 JavaScript 中比较特殊的类型，完美的函数深拷贝几乎不可能实现，因为：
  - 无法拷贝函数的闭包（闭包中的变量是函数内部的私有状态，无法通过外部获取）；
  - 无法拷贝箭头函数的 this 绑定；
  - 无法拷贝 JavaScript 的内置函数（如 console.log、Array.prototype.push）。

因此，在这个实现中：

  - 普通函数：通过 `toString()` 获取函数体，再用 new Function 重建（但闭包会丢失）；
  - 箭头函数：直接返回原函数（因为无法重建 this 绑定）。


2. 循环引用的处理

使用 WeakMap 存储已经拷贝过的对象：
  - 当遇到一个已经拷贝过的对象时，直接返回缓存的拷贝结果，避免无限递归导致栈溢出；
  - WeakMap 的 key 是弱引用，不会影响垃圾回收，避免内存泄漏。

3. 特殊对象的处理
  - Date：通过 `new Date(target.getTime())` 重建；
  - RegExp：通过 `new RegExp(target.source, target.flags) `重建，并保留 lastIndex 状态；
  - Map/Set：遍历并递归拷贝每个 key/value 或 item；
  - Symbol 类型的属性：通过 Object.getOwnPropertySymbols 获取并拷贝。
