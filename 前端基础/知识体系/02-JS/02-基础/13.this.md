# this不同应用场景如何取值


this 的指向不是在定义时确定的，而是在「调用时」确定的 —— **谁调用函数，this 就指向谁。**


## new 调用（构造函数 / 类）→ this 指向新实例

用 new 调用构造函数 / 类时，this 指向新创建的实例对象。


```js
// 构造函数版
function Person(name) {
  this.name = name; // this → new 出来的实例
  console.log(this); // Person { name: '张三' }
}
const p1 = new Person('张三');

// 类版（ES6）
class Student {
  constructor(age) {
    this.age = age; // this → new 出来的实例
  }
}
const s1 = new Student(18);
console.log(s1.age); // 18（this 指向 s1）

```

注意：new 是 this 绑定优先级最高的场景，其他规则都无法覆盖它。



## call/apply/bind 调用 → this 指向指定对象

用 call/apply/bind 显式绑定 this 时，this 指向传入的第一个参数（thisArg）。

```js

const obj = { name: '李四' };
function sayHi() {
  console.log(this.name);
}

// call 绑定
sayHi.call(obj); // 李四（this → obj）
// apply 绑定（和 call 仅传参不同）
sayHi.apply(obj); // 李四
// bind 绑定（返回新函数，this 永久绑定）
const boundSayHi = sayHi.bind(obj);
boundSayHi(); // 李四

```

- call/apply 立即执行，bind 返回新函数（不立即执行）；

- 若传入 null/undefined，this 会指向全局对象（浏览器 window/Node global）。



## 对象方法调用 → this 指向调用方法的对象

函数作为对象的方法被调用时，this 指向这个对象本身（“点前面的对象”）。


```js

const person = {
  name: '王五',
  sayHi: function() {
    console.log(this.name); // this → person
  }
};

// 直接调用对象方法
person.sayHi(); // 王五

// 注意：方法赋值给变量后，this 会变！
const fn = person.sayHi;
fn(); // undefined（此时 fn 是全局调用，this → window）

```

核心是 “谁调用”：person.sayHi() → person 调用 → this 指向 person；

若方法嵌套调用，this 指向最终调用的对象：

```js

const a = {
  name: 'a',
  b: {
    name: 'b',
    sayHi: function() {
      console.log(this.name); // b（this → b）
    }
  }
};

a.b.sayHi(); 

```

## 普通函数调用（全局 / 独立调用）→ this 指向全局对象


普通函数直接调用（无对象、无 new、无 call/apply/bind），this 指向：

- 浏览器：window；

- Node.js：global；

- 严格模式（use strict）：undefined。

```js

// 全局函数
function fn() {
  console.log(this); 
}
fn(); // 浏览器 → window，Node → global

// 严格模式
function strictFn() {
  'use strict';
  console.log(this); // undefined
}
strictFn();

// 立即执行函数（IIFE）
(function() {
  console.log(this); // window/global
})();

```

这是最容易踩坑的场景：比如把对象方法赋值给变量后调用，就会变成普通函数调用；


## DOM 事件 / 定时器调用 → this 指向触发者 / 全局


### DOM 事件处理函数

this 指向触发事件的 DOM 元素（和 event.target 一致，除非手动绑定）。

```js

// HTML：<button id="btn">点击</button>
const btn = document.getElementById('btn');
btn.onclick = function() {
  console.log(this); // <button id="btn">（触发事件的元素）
};
btn.addEventListener('click', function() {
  console.log(this); // 同上
});

```

### 定时器 / 延时器（setTimeout/setInterval）

回调函数的 this 指向全局对象（浏览器 window/Node global），除非手动绑定。

```js
const obj = { name: '赵六' };
setTimeout(function() {
  console.log(this.name); // undefined（this → window）
}, 1000);

// 手动绑定 this（用 bind）
setTimeout(function() {
  console.log(this.name); // 赵六（this → obj）
}.bind(obj), 1000);

```


## 箭头函数 → this 继承外层作用域的 this

箭头函数没有自己的 this，**它的 this 是「定义时」外层作用域的 this（而非调用时）**，且无法被 call/apply/bind 修改。

```js

// 例子 1：对象方法中的箭头函数
const obj = {
  name: '钱七',
  sayHi: () => {
    console.log(this.name); // undefined（this → window，因为外层是全局作用域）
  },
  sayHi2: function() {
    const inner = () => {
      console.log(this.name); // 钱七（this 继承外层 sayHi2 的 this → obj）
    };
    inner();
  }
};
obj.sayHi(); // undefined
obj.sayHi2(); // 钱七

// 例子 2：定时器中的箭头函数（无需 bind）
const obj2 = { name: '孙八' };
setTimeout(() => {
  console.log(this.name); // undefined（外层是全局，this → window）
}, 1000);

obj2.sayHi = function() {
  setTimeout(() => {
    console.log(this.name); // 孙八（继承外层 sayHi 的 this → obj2）
  }, 1000);
};
obj2.sayHi();

```

箭头函数的 this 是 “静态的”，定义时就确定，永不改变；


## 汇总


- 裸调（不带任何前缀） → this 指向全局（非严格）或 undefined（严格）

- 谁调用 → this 就指向谁（点号前面的是谁，this 就是谁）

- 箭头函数没有自己的 this，它永远用定义时外层作用域的 this

- call/apply/bind 出现 → this 被强制指定为第一个参数

- new + 构造函数 → this 指向新创建的实例

- 普通函数做事件回调 → this 是触发事件的元素

- 箭头函数做事件回调 → this 是外层作用域的 this

- 定时器回调是普通函数 → this 指向全局，用箭头函数 → 记住外层 this


::: tip

- new 调用 → this 是新实例
- bind 过 → this 是 bind 绑定的（bind 优先级最高）
- call / apply → this 是第一个参数
- 对象.方法() → this 是点前面的对象
- 普通函数直接调用 / 定时器回调 → this 是全局（非严格）或 undefined（严格）
- 箭头函数 → 没有自己的 this，往外层找（外层是普通函数 → 普通函数的 this，外层是全局 → 全局）

:::


## 常见坑

### 坑点 1：方法赋值给变量后 this 丢失

```js

const obj = {
  name: '坑1',
  fn: function() { 
    console.log(this.name);
  }
};

const fn = obj.fn;
fn(); // undefined（变成普通函数调用）

```

避坑：用 bind 绑定 this → const fn = obj.fn.bind(obj);



### 坑点 2：定时器回调 this 指向全局

```js

const obj = {
  name: '坑2',
  fn: function() {
    setTimeout(function() {
      console.log(this.name); // undefined
    }, 100);
  }
};

```

避坑：用箭头函数 → `setTimeout(() => { console.log(this.name); }, 100)`;

### 坑点 3：箭头函数作为对象方法

```js

const obj = {
  name: '坑3',
  fn: () => { console.log(this.name); }
};

obj.fn(); // undefined（this 指向全局）

```

避坑：对象方法用普通函数，内层用箭头函数。
